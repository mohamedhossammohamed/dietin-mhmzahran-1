# Dietin Project Context (GEMINI.md)

This file provides comprehensive context for AI agents (like Gemini) interacting with the **Dietin** project.

## 🚀 Project Overview
Dietin is a full-featured health and fitness application designed to help users track their nutrition, workouts, and health goals. It features a modern web interface built with React and Vite, wrapped natively for mobile platforms using Capacitor. Additionally, it incorporates a specialized Python-based AI backend for advanced nutrition analysis, computer vision, and multimodal processing.

## 🛠️ Technical Stack
### Frontend & Mobile
*   **Frameworks**: React (TypeScript), Vite
*   **Mobile**: Capacitor (Android/iOS capabilities)
*   **Styling & UI**: Tailwind CSS, Shadcn UI, Framer Motion, Radix UI
*   **State Management**: Zustand (with persistence), React Query
*   **3D Elements**: React Three Fiber, Three.js
*   **Services**: Firebase (Auth, Analytics, Firestore)

### Python Backend (`python_backend/`)
*   **Framework**: FastAPI, Uvicorn
*   **AI/ML Tools**: PyTorch, Hugging Face Transformers, OpenAI Whisper, ChromaDB
*   **Core Engines**: Vision engine (`vision_engine.py`), nutrition engine (`nutrition_engine.py`), and LLM routing.

## 🏗️ Architecture & Directory Structure
*   **`src/`**: Main frontend source code.
    *   `src/components/`: Reusable UI components, AI interaction components, and 3D elements.
    *   `src/lib/`: Core logic, Firebase configuration, AI API logic (`gemini.ts`), and calculations.
    *   `src/stores/`: Zustand state management stores (e.g., `userStore.ts`, `mealStore.ts`).
    *   `src/pages/`: Application views and routing.
*   **`python_backend/`**: Separate FastAPI service handling complex AI/ML workloads.
    *   `main.py`: Entry point for the backend API.
    *   `docs/`: Backend-specific documentation and phase planning.
*   **`docs/`**: Frontend and general project documentation, including architectural analysis.
*   **`dataconnect/` & `dataconnect-generated/`**: Firebase Data Connect schemas and autogenerated SDKs.

## 🏃‍♂️ Building and Running

### Frontend (Web)
*   **Start Dev Server**: `npm run dev`
*   **Build for Production**: `npm run build`
*   **Lint Codebase**: `npm run lint`

### Mobile (Android)
*   **Build & Open in Android Studio**: `npm run build:android`

### Python Backend
*   **Setup**: Navigate to `python_backend/` and run `pip install -r requirements.txt`.
*   **Run Server**: Run the FastAPI application (typically using `uvicorn main:app --reload`).
*   **Testing**: Run tests using `pytest tests/`.

## 📏 Development Conventions
*   **TypeScript**: Maintain strict typing throughout the frontend codebase.
*   **UI/Styling**: Use Tailwind CSS for styling and leverage Shadcn UI components for consistency.
*   **State Management**: Prefer Zustand for client-side state, ensuring proper persistence where necessary. Manage Firebase reads/writes carefully to optimize costs.
*   **AI Integration**: When adding AI features, refer to the prompt patterns in `src/lib/gemini.ts` and ensure any heavy lifting is routed to the Python backend when appropriate.
